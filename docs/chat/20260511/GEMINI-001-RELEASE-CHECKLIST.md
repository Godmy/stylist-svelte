# Готовность к релизу 1.0.1 `stylist-svelte`

Дата: 2026-05-11

## Текущий статус

На основе анализа состояния репозитория и истории переписки от 2026-05-06, проект находится на финальной стадии подготовки патч-релиза `1.0.1`.

### Что уже сделано:

1.  **Версионирование:** В `package.json` зафиксирована версия `1.0.1`.
2.  **Очистка состава пакета:** В поле `files` файла `package.json` добавлены исключения для внутренних доменов (`commerce`, `management`, `social` и др.), тестов и сторибуков.
3.  **Удаление зависимостей:** Полностью удалена зависимость от `lucide-svelte`.
4.  **Реестр иконок:** Реализована внутренняя система иконок на базе SVG-реестра.
5.  **Документация:** Обновлены `README.md` и `CHANGELOG.md`.
6.  **Индексация (режим package):** Проведена очистка `src/lib/index.ts`. Теперь публичный API содержит только разрешенные домены и кластеры (`component`, `const`, `interface`, `type`).

### Выявленные нюансы:

- **SvelteKit-зависимость:** Домен `architecture` (который является публичным) все еще использует импорт `browser` из `$app/environment`. Это вызывает предупреждение при сборке и ограничивает использование библиотеки в чистых Svelte-проектах (без SvelteKit).
- **Форматирование:** Большое количество файлов требует прогона `prettier`.

## Оставшиеся шаги перед публикацией

### 1. Финализация публичного API

- [x] Прогнать индексатор в режиме `package`: `python stylist/indexation/cli.py --generation-mode package`. (Выполнено в рамках текущей сессии).
- [ ] Проверить, что в `src/lib/index.ts` не осталось экспортов из исключенных доменов (например, `commerce`).

### 2. Устранение предупреждений сборки

- [ ] В файле `src/lib/architecture/component/organism/presentation-workspace/index.svelte` заменить:
  ```typescript
  import { browser } from '$app/environment';
  ```
  на более универсальную проверку:
  ```typescript
  const browser = typeof window !== 'undefined';
  ```
  Это сделает библиотеку полностью независимой от SvelteKit.

### 3. Форматирование и валидация

- [ ] Выполнить `yarn format` для приведения всего кода к единому стилю.
- [ ] Выполнить `yarn validate` и убедиться в отсутствии ошибок (Exit Code 0).

### 4. Финальная проверка состава пакета

- [ ] Выполнить `npm pack --dry-run` и внимательно изучить список файлов в `dist`.
- [ ] Убедиться, что `unpackedSize` соответствует ожиданиям (после удаления лишних доменов и `lucide`).

### 5. Публикация

- [ ] Запустить `npm publish` (или `yarn publish`).
- [ ] Проверить работоспособность установленного пакета в пустом проекте (Smoke Test).
