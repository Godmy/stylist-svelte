# Валидация режимов индексации

Дата: 2026-05-06

## Цель

Проверка корректности работы индексатора в разных режимах (`full` vs `package`).

## Режим `package`

В этом режиме индексатор должен:

1. Исключать внутренние домены (`chat`, `commerce`, `playground` и др.).
2. Исключать кластеры `data`, `class`, `function` из экспорта `index.ts`.
3. Сохранять работоспособность компонентов в `dist`.

## Результаты тестов

- [x] Экспорты в `src/lib/index.ts` очищены от приватных доменов.
- [x] Приватные кластеры не попадают в публичные экспорты.
- [x] Относительные пути внутри компонентов в `dist` остаются корректными.

## Риски

- Зависимости некоторых компонентов от `function/state` могут привести к ошибкам, если пользователь захочет создать состояние вручную, не имея доступа к экспорту функции. Решено: на этапе 1.0.1 это допустимо, так как API стейтов еще стабилизируется.
