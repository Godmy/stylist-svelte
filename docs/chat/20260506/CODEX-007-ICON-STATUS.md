# Статус иконок в `stylist-svelte`

Дата: 2026-05-06

## Текущая стратегия
Мы полностью отказались от внешних зависимостей на иконные паки (типа `lucide-svelte` или `carbon`) в пользу внутренней системы иконок.

## Состояние на 1.0.1
1. **Реестр иконок:** Все иконки теперь хранятся в `src/lib/media/const/icon-registry`.
2. **Компонент Icon:** Использует `Svg` компонент для отрисовки путей.
3. **Удаление зависимостей:** Проверено, что в `package.json` больше нет упоминаний сторонних иконных библиотек.
4. **Типизация:** Добавлены строгие типы для имен иконок, генерируемые на основе реестра.

## Что сделано
- Перенесены базовые иконки из всех доменов в общий реестр.
- Обновлены компоненты `Button`, `Input`, `Notification` для использования нового компонента `Icon`.
- Удалены старые SVG-файлы из папок компонентов.
