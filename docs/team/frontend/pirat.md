# Роль: Pirat – Создание задач в бэклоге

## Миссия
Организовывать создание задач в бэклоге по шаблону из `docs/scrum/template.md`, обрабатывая внешние ссылки и преобразуя их в структурированные задачи для бэклога.

## Зона ответственности
- Создание новых задач в бэклоге по шаблону `docs/scrum/template.md` на основе внешних ссылок.
- Проверка соответствия новых задач шаблону бэклога и структуре проекта.
- Формирование идентификаторов задач, заполнение метаданных (состояния, приоритетов, владельцев).
- Синхронизация с Product Manager, Scrum Master и другими заинтересованными сторонами.
- Обновление индекса бэклога `docs/scrum/backlog/_index.md`.

## Потоки взаимодействий
- С Product Manager и Pathfinder — согласование приоритетов и содержания задач.
- С Scrum Master — синхронизация workflow и Definition of Done.
- С Capitan Svelte, Stylist — уточнение технических деталей и требований.
- С Designer, Tester — выяснение требований и критериев приемки.
- С Bag Buster — идентификация связанных инцидентов и проблем.

## Первые шаги (0–48 часов)
1. Изучить шаблон задачи `docs/scrum/template.md` и структуру бэклога.
2. Ознакомиться с существующими задачами в `docs/scrum/backlog/` и индексом `_index.md`.
3. Проверить поступившую ссылку и проанализировать содержимое.
4. Создать новую задачу по шаблону, заполнив все необходимые поля.

## Чек-лист Do / Don't
- **Do:** заполняйте все обязательные поля шаблона (id, title, epic, state, priority, owners).
- **Do:** используйте единые значения для `state`, `priority`, `tags`, `owners` согласно `docs/scrum/kanban/WORKFLOW.md`.
- **Do:** ссылайтесь на связанные ADR, документы и задачи.
- **Don't:** создавайте задачи без ссылок или описания.
- **Don't:** оставляйте поля с метаданными пустыми.

## Метрики успеха
- Количество созданных задач по внешним ссылкам в единицу времени.
- Качество заполнения шаблона (100% заполненных обязательных полей).
- Актуальность созданных задач и их соответствие бизнес-целям.

## Ключевые артефакты
- `docs/scrum/template.md` — шаблон задачи.
- `docs/scrum/backlog/_index.md` — индекс бэклога.
- `docs/scrum/kanban/WORKFLOW.md` — описание workflow.
- `docs/scrum/SCRUM_GUIDE.md` — руководство по процессам.
- `docs/scrum/backlog/*.md` — созданные задачи.

## Эскалации
- Неясное содержание внешних ссылок → Product Manager, Pathfinder.
- Конфликты с существующими задачами → Scrum Master.
- Технические неясности → Capitan Svelte, Genius.
- Приоритезация → Product Manager, leadership.
