# Командная структура проекта

Добро пожаловать в раздел описания командной структуры проекта. Здесь содержатся описания ролей, обязанностей и процессов взаимодействия внутри команды.

## Структура директории

```
team/
├── README.md                   # Этот файл
├── frontend/                   # Роли фронтенд команды
│   ├── capitan.md              # Frontend Lead (Capitan Svelte)
│   ├── stylist.md              # Дизайн-система (Stylist)
│   ├── pirat.md                # Создание задач в бэклоге (Pirat)
│   └── ...
├── management/                 # Роли управления и архитектуры
│   ├── master.md               # Scrum Master (Master)
│   ├── constructor.md          # Архитектор FSD и Atomic Design (Constructor)
│   └── ...
└── ...
```

## Назначение

Эта директория предназначена для:
- Четкого определения ролей и обязанностей участников команды
- Обеспечения прозрачности взаимодействия между членами команды
- Упрощения онбординга новых участников
- Стандартизации процессов управления проектом

## Типы ролей

### Frontend команды
- **Capitan Svelte**: Frontend Lead, отвечает за архитектуру фронтенда
- **Stylist**: Специалист по дизайн-системе, отвечает за компонентную архитектуру
- **Pirat**: Отвечает за создание задач в бэклоге по внешним ссылкам

### Управленческие и архитектурные роли
- **Master**: Scrum Master, фасилитирует процессы и устраняет блокеры
- **Constructor**: Архитектор, отвечает за FSD и Atomic Design архитектуру

## Основные принципы

- Каждая роль имеет четко определенные обязанности и зону ответственности
- Роли могут пересекаться для обеспечения эффективного взаимодействия
- Описание ролей обновляется при изменении процессов или структуры команды
- Все участники команды должны понимать, как их роль взаимодействует с другими

## Для новых участников

Если вы новичок в проекте:
1. Изучите описание вашей роли в соответствующем файле
2. Ознакомьтесь с `../scrum/README.md` для понимания процессов
3. Проверьте, как ваша роль взаимодействует с другими ролями
4. Обратитесь к руководителю команды или Scrum Master для онбординга
