# ADR 0009: Yarn 4 как стандарт управления пакетами

**Status:** Accepted (Revised)
**Date:** 2025-01-03
**Revised:** 2026-02-20

## Контекст
Проект использует Yarn 4, но не является активным multi-workspace monorepo в текущем состоянии.

## Решение
Фиксируем решение: стандартный пакетный менеджер проекта — Yarn 4 (`yarn@4.5.1`).

## Проверка по кодовой базе
- `packageManager: yarn@4.5.1` указан в `stylist-svelte/package.json`.
- Единый `package.json` и `yarn.lock` используются как основной контур сборки.

## Последствия
- Единообразные команды сборки/проверок.
- Предсказуемое поведение dependency management.
