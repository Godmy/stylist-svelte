# ADR 0003: Компонентная архитектура по Atomic Design

**Status:** Accepted (Validated)
**Date:** 2025-01-03
**Validated:** 2026-02-20

## Контекст
Библиотека содержит сотни компонентных единиц и требует устойчивой таксономии.

## Решение
Базовый порядок организации фиксируем так:
1. Сначала Atomic Design деление: `atoms`, `molecules`, `organisms`.
2. Затем функциональная таксономия внутри уровня: `architecture`, `information`, `interaction`.
3. Затем доменная кластеризация внутри функциональной таксономии: `controls`, `chat`, `media` и т.п.

## Проверка по кодовой базе
- Структура присутствует в `src/lib/components/{atoms,molecules,organisms}`.
- Внутри уровней используется функциональная таксономия (`architecture/information/interaction`) как второй слой.
- Внутри функциональной таксономии используется доменная кластеризация (например, `controls`, `chat`, `media`) как третий слой.

## Последствия
- Ясные границы ответственности и зависимостей.
- Упрощение навигации и ревью изменений.
- Стабильный приоритет структурирования: Atomic Design -> функциональная таксономия -> доменная кластеризация.
