# ADR 0004: Собственная Playground-система

**Status:** Accepted (Validated)
**Date:** 2025-01-03
**Validated:** 2026-02-20

## Контекст
Нужна встроенная среда демонстрации и проверки компонентов, тесно связанная с библиотекой.

## Решение
Используем собственный Playground-слой внутри проекта вместо внешнего монолитного инструмента.

## Проверка по кодовой базе
- Есть `src/lib/design-system/playground`.
- Совместимость экспортируется через `src/lib/playground/index.ts`.
- Компонентные story-файлы (`*.story.svelte`) широко используются в `src/lib/components/**`.

## Последствия
- Гибкий контроль над DX и структурой историй.
- Единая экосистема внутри репозитория.
