# ADR 0008: Композитная архитектура таблиц

**Status:** Accepted (Revised)
**Date:** 2025-01-04
**Revised:** 2026-02-20

## Контекст
Табличные сценарии требуют composable-подхода (table/header/body/row/cell + расширения).

## Решение
Поддерживаем таблицы как набор композиционных компонентов и контрактов:
- UI-секции в компонентах `atoms/molecules/organisms`;
- типы и стили в `design-system/props/table*.ts` и `design-system/styles/table*.ts`.

## Проверка по кодовой базе
- `src/lib/types/table.ts` отсутствует как отдельный источник правды.
- Табличные контракты и стили фактически живут в design-system.

## Последствия
- Более гибкая эволюция таблиц без централизованного «узкого» файла типов.
- Устойчивый путь для feature-расширений.
