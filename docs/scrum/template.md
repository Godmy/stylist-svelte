# Практики управления бэклогом

## Единый индекс
- Ведите `packages/backend/docs/scrum/backlog/_index.md` с таблицей (ID, Заголовок, Состояние, Приоритет, Ответственные, Обновлено).
- Генерируйте JSON/CSV из front matter, чтобы ИИ-агенты могли фильтровать и приоритизировать программно.

## Кодировка и формат
- Используйте UTF-8 для всех документов; избегайте устаревших кодировок и псевдографики.
- Нормализуйте использование `state`, `priority`, `tags`, `owners` и `epic` для согласованного поиска и аналитики.

## Отслеживаемость
- Ссылайтесь на ADR/FDD/FSD документы в `linked_adr` и поддерживайте их в актуальном состоянии.
- Заполняйте `dependencies`, чтобы агенты понимали порядок и блокирующие работы.

## Набор полей для ИИ
- Обновляйте `ai_ready.summary`, `entry_points`, `expected_output` всякий раз, когда изменяется объем работ.
- Фиксируйте каноничные команды и артефакты, чтобы агент мог выполнить задачу от начала до конца.

## Контроль качества
- Применяйте четкие чек-листы Definition of Ready/Done для каждой команды.
- Добавьте линтер пред-коммита/CI, который проверяет обязательные поля во front matter.
- Проверяйте `state` и `priority` по графику, чтобы закрывать или переприоритизировать устаревшие элементы.
