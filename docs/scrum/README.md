# SCRUM Документация

Добро пожаловать в раздел документации по SCRUM-процессам проекта. Здесь содержатся все шаблоны, руководства и шаблоны задач, необходимые для эффективного управления разработкой по гибким методологиям.

## Структура директории

```
scrum/
├── template.md                 # Шаблон создания задач в бэклоге
├── SCRUM_GUIDE.md              # Основное руководство по SCRUM процессам
├── README.md                   # Этот файл
├── kanban/                     # Документация по работе с Kanban доской
│   ├── WORKFLOW.md             # Описание рабочего процесса и состояний задач
│   └── _index.md               # Индекс задач бэклога
└── backlog/                    # Задачи в бэклоге проекта
    ├── _index.md               # Общий индекс задач
    └── *.md                    # Отдельные задачи (названия в формате ID-описание.md)
```

## Назначение

Эта директория предназначена для:
- Стандартизации процессов управления задачами
- Обеспечения прозрачности разработки
- Упрощения интеграции новых участников команды
- Автоматизации управления бэклогом с помощью ИИ-агентов

## Основные принципы

- Все задачи создаются по шаблону `template.md` для обеспечения полноты информации
- Состояния задач строго регламентированы в `kanban/WORKFLOW.md`
- Каждая задача содержит обязательные метаданные для автоматического анализа
- Документация поддерживается в актуальном состоянии всеми участниками команды

## Для новых участников

Если вы новичок в проекте:
1. Ознакомьтесь с `SCRUM_GUIDE.md` для понимания процессов
2. Изучите `template.md` чтобы понимать, как создаются задачи
3. Проверьте `kanban/WORKFLOW.md` чтобы понимать жизненный цикл задачи
4. Обратитесь к README в соответствующей команде из `../team/` для понимания ваших обязанностей
