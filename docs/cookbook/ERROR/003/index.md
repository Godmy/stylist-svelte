# ERROR-003: Реэкспорт запрещён

**Серьёзность:** ❌ Error  
**Правило ADR:** Запрещено делать реэкспорт и тем более держать для этих целей целый файл — это размывает границы доменов и мешает связности.

---

## Что произошло

Скрипт нашёл `export { ... }` или `export * from ...` в файле, который не является баррелем.  
Такой файл занимает место сущности, но сам ничего не определяет — только перенаправляет.

---

## Примеры

```ts
// ❌ Реэкспорт из другого модуля
export { SomeType } from '../other-domain/type/struct/some-type';

// ❌ Реэкспорт всего
export * from './helpers';

// ❌ Переименовывающий реэкспорт
export { Foo as Bar } from './foo';
```

---

## Как исправить

### Случай 1 — Реэкспорт ради удобного импорта

Удалить файл. Импортировать напрямую из исходного домена.

```ts
// ✅ Импортировать в потребителе напрямую:
import type { SomeType } from '@lib/other-domain/type/struct/some-type';
```

### Случай 2 — Сущность живёт не в том домене

Если реэкспорт появился потому, что тип лежит «не там», — переместить тип в правильный домен и обновить импорты.

### Случай 3 — Баррель, написанный вручную

Если файл содержит `export * from ...` и это намеренный баррель — баррель должен генерироваться автоматически, не редактируется вручную.  
Удалить содержимое, пересобрать.
