# utils

## Целесообразность папки
Папка оправдана как слой общих функций, не принадлежащих конкретному компонентному домену.

## Характеристика домена
Домен утилит: class merging, helper-классы input, операции с CSS variables.

## Типовой функционал, поведение или признак агрегации
Типовое поведение: преобразования и вспомогательные вычисления без хранения состояния.

Агрегация по принципу cross-cutting concerns: функции используются разными доменами.

## Способ консолидации информации
Консолидация через utils/index.ts и подпапку css-vars для тематических утилит.

## Признаки системы
Системные признаки: чистые функции, повторное использование, снижение дублирования.

## SOLID через призму SAMO
SAMO трактуется как: S (Scope/границы ответственности), A (Abstractions/контракты), M (Modularity/композиция), O (Operability/эксплуатация и поддержка).

- SRP через S: папка держит одну главную ответственность внутри своего домена.
- OCP через A+M: расширение идет добавлением новых файлов/экспортов, а не переписыванием существующих контрактов.
- LSP через A: типовые контракты и сигнатуры позволяют безопасно подставлять реализации.
- ISP через S+M: API дробится на небольшие модули, потребители берут только нужные части.
- DIP через A+O: код зависит от типов/контрактов и фабричных/утилитарных абстракций, а не от конкретных реализаций.
