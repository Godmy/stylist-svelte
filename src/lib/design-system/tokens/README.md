# tokens

## Назначение
`tokens` — источник атомарных значений дизайн-системы.

## Что хранится
- Цвета, размеры, spacing, radius, z-index, variants, flags.
- Только декларации значений и типов.

## Что не хранится
- Runtime state-логика.
- Class maps.
- Component presets/defaults.

## Таксономия
Используется разделение:
- `architecture`
- `information`
- `interaction`

## Правила расширения
1. Добавляйте токены только как переиспользуемые primitives.
2. Избегайте токенов «под один компонент» без явной причины.
3. Предпочитайте типизированные словари и `as const`.
