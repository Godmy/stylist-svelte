# styles

## Назначение
`styles` — слой `*StyleManager`, собирающий итоговые классы для компонентов.

## Роль в цепочке
`styles` использует `classes`/`tokens`/`props`, но не заменяет их.

## Что допустимо
- Композиция классов.
- Нормализация style-вариантов.
- Возврат готовых строк/структур классов.

## Что недопустимо
- Бизнес-правила предметной области.
- Runtime state orchestration (это слой `state/models`).

## Эффективность
Подход хорошо масштабируется по числу компонентов, потому что логика стилей изолирована.

## Риски
Если `styles` начинает дублировать class maps вместо импорта из `classes`, растёт технический долг.
