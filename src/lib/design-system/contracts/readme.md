# contracts

## Назначение
`contracts` — типизированный контракт для компонентов.

## Смысл
- Компоненты и модели не должны придумывать ad-hoc интерфейсы на месте.
- Контракт хранится централизованно и версионируется как API.

## Таксономия
Файлы организованы по доменам:
- `architecture`
- `information`
- `interaction`

## Правила расширения
1. Добавляйте новые типы в домен компонента.
2. Ломающее изменение в публичном типе — только осознанно.
3. Дубли типов между доменами выносите в общее ядро.
