# QWEN.md - Инструкциональный контекст для проекта stylist-svelte

## Обзор проекта

**Название**: @stylist-svelte
**Тип**: Библиотека Svelte компонентов
**Версия**: 0.0.1
**Описание**: Повторно используемая библиотека Svelte компонентов, построенная на Svelte 5 с полной системой документации Histoire.

### Особенности:
- **Svelte 5**: Использует последние возможности Svelte, включая runes
- **Поддержка TypeScript**: Полная типизация
- **Документация Histoire**: Интерактивный плейграунд для компонентов
- **Atomic Design Architecture**: Компоненты организованы в соответствии с принципами атомического дизайна (атомы, молекулы, организмы)
- **Доступность**: Разработано с учетом лучших практик a11y
- **Адаптивность**: Дизайн с приоритетом мобильных устройств

## Сборка и запуск

### Предварительные требования:
- Node.js (v18 или выше)
- npm, yarn или pnpm

### Установка:
1. Установить зависимости:
   ```bash
   yarn install
   ```

### Доступные скрипты:
- `yarn dev` - Запуск сервера разработки Histoire
- `yarn build` - Сборка библиотеки компонентов
- `yarn preview` - Предварительный просмотр собранной документации
- `yarn check` - Проверка типов Svelte
- `yarn lint` - Проверка кода
- `yarn format` - Форматирование кода с помощью Prettier
- `yarn test` - Запуск тестов
- `yarn test:run` - Запуск тестов без наблюдения
- `yarn coverage` - Покрытие тестами

### Запуск разработки:
Чтобы начать работу с проектом, запустите:
```bash
yarn dev
```
Это запустит интерактивный сервер документации Histoire, где можно просматривать и тестировать компоненты.

## Конфигурация

### svelte.config.js
Файл конфигурации Svelte, который использует vitePreprocess() и имеет настройку compilerOptions с customElement: false.

### histoire.config.ts
Файл конфигурации Histoire, который настраивает плагин Svelte, тему, логотипы, параметры отображения историй и адаптивные пресеты.

### package.json
Важные зависимости:
- svelte: ^5.0.0
- @sveltejs/vite-plugin-svelte
- @sveltejs/package
- @sveltejs/kit
- histoire и @histoire/plugin-svelte
- Типы и инструменты для TypeScript, ESLint и Prettier

## Структура компонентов

В проекте используется архитектурный подход Atomic Design:
- **Атомы** (atoms): Простые базовые компоненты (кнопки, инпуты, лейблы)
- **Молекулы** (molecules): Комбинации атомов (формы, карточки)
- **Организмы** (organisms): Комплексные структуры из молекул и атомов (навигация, шапка сайта)

## Правила разработки

### Создание новых компонентов:
1. Размещайте в соответствующем каталоге (atoms/molecules/organisms)
2. Создайте соответствующую историю в каталоге `stories/`
3. Экспортируйте из соответствующего файла `index.ts`
4. Добавьте правильные TypeScript типы
5. Следуйте существующему стилю кода

### Тестирование:
Для запуска тестов используйте команды:
- `yarn test` - для запуска в режиме разработки с наблюдением
- `yarn test:run` - для однократного запуска тестов

## Публикация

Для публикации новой версии:
1. Обновите версию в package.json, следуя семантическому версионированию
2. Запустите тесты: `yarn test`
3. Соберите библиотеку: `yarn build`
4. Опубликуйте в npm: `yarn publish`

Для предварительных релизов:
```bash
yarn version prerelease --preid=beta
yarn publish --tag beta
```
